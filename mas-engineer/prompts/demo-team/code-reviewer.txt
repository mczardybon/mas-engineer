Build a complete Multi-Agent System called "code-reviewer" at
/tmp/code-reviewer. Do NOT just plan it — actually create the files,
run live tests, and report results.

ARCHITECTURE: Parallel fan-out then aggregator (4 agents)

  code input
      |
      v
  review-orchestrator
      |
      +-- parallel start -----------------------+
      |                                         |
      +-> style-reviewer     (lint, formatting) |
      |                                         |
      +-> security-reviewer  (vulns, secrets)   |
      |                                         |
      +-> perf-reviewer      (perf, complexity) |
      |                                         |
      +-> test-reviewer      (coverage, edge)   |
      |                                         |
      parallel end (all 4 results) --------------+
      |
      v
  review-aggregator   (merges + prioritizes)
      |
      v
  final review report


STEP 1 — Initialize the project skeleton
Run: python3 ~/.config/goose/recipes/tools/dev_generic_init.py
     --init /tmp/code-reviewer --components all

STEP 2 — Create 4 parallel agents + 1 aggregator in
         /tmp/code-reviewer/recipe/sub/:

1. style-reviewer.yaml
   - Reviews code for: lint errors, formatting, naming conventions,
     import order, dead code
   - Returns: {issues: [{file, line, severity, message, suggestion}]}

2. security-reviewer.yaml (MANDATORY — security gate)
   - Reviews code for: hardcoded secrets, SQL injection, XSS,
     insecure dependencies, missing auth checks
   - Returns: {issues: [{file, line, severity, cwe_id, fix}]}
   - This is a hard quality gate. Any HIGH severity issue
     blocks the merge.

3. perf-reviewer.yaml
   - Reviews code for: O(n^2) loops, blocking I/O, memory leaks,
     redundant computation, missing caching
   - Returns: {issues: [{file, line, severity, impact_ms, fix}]}

4. test-reviewer.yaml
   - Reviews code for: missing tests, edge cases, mock coverage,
     flaky patterns
   - Returns: {issues: [{file, line, severity, missing_scenarios}]}

5. review-aggregator.yaml
   - Receives {style_issues, security_issues, perf_issues, test_issues}
   - Merges into a single prioritized list (critical, high, medium, low)
   - Calculates overall score (0-100)
   - Decides: approve | request_changes | needs_discussion
   - Returns: {score, decision, summary, all_issues_sorted}

STEP 3 — Create orchestrator root recipe
/tmp/code-reviewer/recipe/code-reviewer.yaml that runs the 4 reviewers
in PARALLEL (no dependencies between them) and then dispatches to
review-aggregator. Document the parallel execution in instructions.
