Build a complete Multi-Agent System called "data-analyzer" at
/tmp/data-analyzer. Do NOT just plan it — actually create the files,

ARCHITECTURE: Iterative loop with convergence check (6 agents)

  dataset input
      |
      v
  analysis-loop-controller   (runs up to N iterations)
      |
      +-> iteration 1, 2, ... N --------------------+
      |                                              |
      +-> data-profiler                             |
      |   (schema, types, missing values, stats)    |
      |       |                                      |
      |       v                                      |
      |   data-quality-checker                       |
      |   (anomalies, outliers, drift)               |
      |       |                                      |
      |       v                                      |
      |   insight-generator                          |
      |   (hypotheses, correlations, patterns)       |
      |       |                                      |
      |       v                                      |
      |   critic-reviewer                            |
      |   (verifies insights, looks for false pos)  |
      |       |                                      |
      |       v                                      |
      |   convergence-checker                        |
      |   (have insights stopped changing?)          |
      |       |                                      |
      |       v                                      |
      |   <loop back to data-profiler if not done>  |
      |                                              |
      +----------------------------------------------+
      |
      v
  final-report-compiler      (assembles all iterations)
      |
      v
  report to user


STEP 1 — Initialize the project skeleton
Run: python3 ~/.config/goose/recipes/tools/dev_generic_init.py
     --init /tmp/data-analyzer --components all

STEP 2 — Create 6 agents in /tmp/data-analyzer/recipe/sub/:

1. analysis-loop-controller.yaml
   - Orchestrator. Tracks iteration count (max 3).
   - Receives {dataset_description, max_iterations=3}
   - Loops: profile → quality → insight → critic → convergence
   - If convergence says "stable" OR max reached → exit loop
   - Calls final-report-compiler
   - Returns: {iterations: [...], final_report}

2. data-profiler.yaml
   - Input: {dataset_description, previous_profile? (for diff)}
   - Output: {schema, types, row_count, missing_pct, numeric_stats,
             categorical_cardinality, sample_rows}
   - On iteration > 1: also returns {delta_from_previous}

3. data-quality-checker.yaml
   - Input: {profile}
   - Output: {duplicates, outliers, null_patterns, drift_score,
             quality_score: 0-100}

4. insight-generator.yaml
   - Input: {profile, quality}
   - Output: {hypotheses: [{claim, confidence, evidence}],
              correlations: [{vars, r, p}],
              patterns: [str]}

5. critic-reviewer.yaml
   - Input: {insights, profile, quality}
   - Output: {verified: [int], rejected: [{claim, reason}],
              refined: [{claim, refinement}],
              confidence_adjustments: {claim: delta}}

6. convergence-checker.yaml
   - Input: {previous_insights, current_insights, iteration}
   - Output: {stable: bool, change_pct: float, recommendation:
              "continue" | "stop_converged" | "stop_stalled"}

7. final-report-compiler.yaml
   - Input: {iterations, final_insights}
   - Output: {executive_summary, key_findings, recommendations,
              methodology, iterations_log}

STEP 3 — Create orchestrator root recipe
/tmp/data-analyzer/recipe/data-analyzer.yaml that references the
controller as a sub_recipe. Controller internally references the
6 other agents. Maximum 3 iterations. Loop terminates on convergence.
