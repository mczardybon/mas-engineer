Build a complete Multi-Agent System called "customer-support" at
/tmp/customer-support. Do NOT just plan it — actually create the files,
run live tests, and report results.

ARCHITECTURE: Linear pipeline (3 agents, sequential flow)

  user message
      |
      v
  intent-router        (classifies: billing|tech|general)
      |
      v
  specialist-handler   (handles the actual issue)
      |
      v
  empathic-responder   (adds empathy, formats final reply)


STEP 1 — Initialize the project skeleton
Run: python3 ~/.config/goose/recipes/tools/dev_generic_init.py
     --init /tmp/customer-support --components all

STEP 2 — Create 3 agents in /tmp/customer-support/recipe/sub/:

1. intent-router.yaml
   - Reads user message
   - Classifies into one of: "billing", "technical", "general"
   - Returns: {category: str, confidence: float, reasoning: str}
   - If confidence < 0.6, falls back to "general"

2. specialist-handler.yaml (sub_recipes based on category)
   - Receives {category, user_message}
   - Dispatches to the matching handler:
     * billing-handler: handles invoices, refunds, subscriptions
     * tech-handler: handles bugs, errors, integrations
     * general-handler: handles questions, feedback
   - Returns: {solution: str, steps: [str], needs_escalation: bool}

3. empathic-responder.yaml
   - Receives {user_message, solution, steps}
   - Adds empathy (acknowledges frustration if any)
   - Formats final reply as: greeting + empathy + solution + steps + offer
   - Returns: final customer-facing response string

STEP 3 — Create orchestrator root recipe /tmp/customer-support/recipe/
customer-support.yaml that references all 3 sub-recipes via sub_recipes.
The orchestrator runs them in strict sequence:
intent-router → specialist-handler → empathic-responder
