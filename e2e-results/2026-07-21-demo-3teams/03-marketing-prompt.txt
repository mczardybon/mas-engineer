Build a complete Multi-Agent System called "marketing-team" at
/tmp/marketing-team using a HUB-AND-SPOKE architecture. Do NOT just
plan it — actually create the files, run live tests, and report results.

ARCHITECTURE (HUB-AND-SPOKE — IMPORTANT):
The marketing-orchestrator is the HUB. It receives any marketing query
and DYNAMICALLY decides which specialist(s) to invoke. There is NO
fixed pipeline order. Each specialist can be called independently,
in parallel, or skipped entirely based on the query.

STEP 1 — Initialize the project skeleton
Run: python3 ~/.config/goose/recipes/tools/dev_generic_init.py
     --init /tmp/marketing-team --components all

STEP 2 — Create 5+ agents in /tmp/marketing-team/recipe/sub/:

1. marketing-orchestrator.yaml (root: /tmp/marketing-team/recipe/
   marketing-team.yaml)
   - HUB. Receives any marketing query.
   - Classifies intent into one or more:
     * seo → call seo-researcher
     * content → call content-writer
     * social → call social-media-manager
     * analytics → call analytics-reporter
     * email → call email-campaign-manager
   - DYNAMICALLY dispatches: can call 1, 2, 3, or all 5 in parallel.
   - NO fixed order. NO mandatory pipeline. Orchestrator decides.
   - Collects results and synthesizes a unified marketing plan.

2. seo-researcher.yaml
   - Uses web_search to find SEO keywords, competitor analysis,
     search volume data.
   - Returns: list of {keyword, volume, difficulty, source_url}.

3. content-writer.yaml
   - Receives brief (topic + SEO keywords if available).
   - Writes blog post, landing page copy, or ad copy.
   - Returns {title, body, meta_description, cta}.

4. social-media-manager.yaml
   - Receives content (or topic).
   - Adapts to 3-4 platforms (Twitter/X, LinkedIn, Instagram, TikTok).
   - Returns {platform, post_text, hashtags, source_url}.

5. analytics-reporter.yaml
   - Uses web_search to find marketing benchmarks, case studies,
     industry stats.
   - Returns {metric, value, source_url, confidence}.

6. email-campaign-manager.yaml
   - Receives content + audience.
   - Drafts email sequence (subject + body + send-time).
   - Returns {email_n, subject, body, send_time, cta}.

STEP 3 — Create the orchestrator root recipe marketing-team.yaml
that references all 5 sub-recipes via sub_recipes field.
