ARCHITECTURE (PARALLEL + VOTING — IMPORTANT):
The translator-orchestrator receives a source text + target language.
It PARALLEL dispatches the SAME text to 3 different translator agents
(translator-literal, translator-literary, translator-technical) at
the SAME TIME. Each produces an independent translation. Then a
JUDGE agent (translation-judge) compares all 3 outputs and votes for
the best one based on accuracy, fluency, and style. The judge returns
the winning translation with its reasoning.

This is NOT a pipeline. It is NOT hub-and-spoke. It is PARALLEL
generation followed by a VOTE/SELECT step.

STEP 1 — Initialize the project skeleton
Run: python3 ~/.config/goose/recipes/tools/dev_generic_init.py
     --init /tmp/translator-team --components all

STEP 2 — Create 5 agents in /tmp/translator-team/recipe/sub/:

1. translator-orchestrator.yaml (root: /tmp/translator-team/recipe/
   translator-team.yaml)
   - Receives {source_text, source_lang, target_lang}.
   - PARALLEL dispatches the same text to all 3 translator agents
     simultaneously (NOT sequentially).
   - After all 3 return, delegates to translation-judge with all 3
     candidate translations.
   - Returns the judge's chosen translation with vote reasoning.

2. translator-literal.yaml
   - Style: word-for-word literal translation.
   - Preserves technical terms, no idiomatic substitutions.
   - Returns {style: "literal", text, notes}.

3. translator-literary.yaml
   - Style: literary, idiomatic, preserves tone and emotion.
   - Adapts metaphors and cultural references.
   - Returns {style: "literary", text, notes}.

4. translator-technical.yaml
   - Style: technical, preserves domain jargon and precision.
   - Keeps numbers, units, formulas exact.
   - Returns {style: "technical", text, notes}.

5. translation-judge.yaml
   - Receives the 3 candidate translations + source text.
   - Scores each on accuracy (0-1), fluency (0-1), style-fit (0-1).
   - VOTES for the best one.
   - Returns {winner, scores, reasoning, final_text}.

STEP 3 — Create the orchestrator root recipe translator-team.yaml
that references all 4 sub-recipes via sub_recipes field.
